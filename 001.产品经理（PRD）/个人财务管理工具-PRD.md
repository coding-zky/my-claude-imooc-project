## 一、产品概述

### 1.1 产品定位

一款极简的个人日常记账工具，帮助用户养成每日记录消费的习惯，清晰掌握自己的资金流向，做到「花出去的每一笔钱，心里都有数」。

### 1.2 目标用户

| 用户画像 | 特征 |
|---------|------|
| 职场新人 | 刚工作 1-3 年，开始有理财意识，想搞清楚钱花在哪了 |
| 月光族 | 月底总是没钱，想通过记账找到消费漏洞 |
| 极简记账党 | 不想用复杂的记账软件，只要快速记一笔就行 |

### 1.3 核心价值主张

**3 秒记一笔** — 打开即记，记完即走，零学习成本。

### 1.4 产品愿景

从「记录」到「理解」再到「改善」，分三步走：
- **V1.0（记录）**：让用户能快速、持续地记下来
- **V2.0（理解）**：让用户看清楚钱去哪了
- **V3.0（改善）**：帮用户做出更好的消费决策

## 三、详细功能说明

### 3.1 快速记账（核心功能）

**交互流程**：

```
点击首页 [+] 按钮
    ↓
弹出记账面板（底部弹窗）
    ↓
┌──────────────────────────┐
│  ○ 支出    ○ 收入         │  ← 类型切换
│                          │
│  [      0.00      ]      │  ← 金额输入（大字号，自动聚焦）
│                          │
│  分类：[🍔 餐饮      ▼]   │  ← 分类选择（默认上一次）
│  日期：[2026-05-13  ▼]   │  ← 日期（默认今天）
│  备注：[添加备注...    ]  │  ← 可选
│                          │
│  [       记一笔         ] │  ← 确认按钮
└──────────────────────────┘
```

**设计要点**：
- 打开面板时自动弹出数字键盘，光标已在金额输入框
- 分类默认选中上一次使用的分类
- 日期默认今天，支持回溯补记
- 点击「记一笔」后面板关闭，首屏数据即时刷新，并给予轻反馈（震动/Haptic）
- 支持连续记账模式：记完一笔后不清空分类和类型，仅清空金额，方便连续录入

### 3.3 统计页

**排行区域**：按金额从高到低列出各分类，右侧显示金额和占比进度条。

### 3.6 登录与注册（新增 V1.0.1）

由于本产品定位为**纯本地应用**（数据完全存储在浏览器本地，不上传任何服务器），登录/注册功能采用本地账号体系，无需联网即可使用。

#### 3.6.1 首次使用 — 注册

```
┌──────────────────────────────┐
│                              │
│        🏦 记账本              │
│     个人财务管理专家           │
│                              │
│  ┌────────────────────────┐  │
│  │ 昵称：[请输入你的昵称]   │  │  ← 必填，用于首页问候
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │ 头像：[预设头像列表]     │  │  ← 可选，6 个预设头像
│  └────────────────────────┘  │
│  ┌────────────────────────┐  │
│  │ PIN 码：[····]          │  │  ← 可选，4-6 位数字
│  └────────────────────────┘  │
│                              │
│  ┌────────────────────────┐  │
│  │     开始使用 →          │  │
│  └────────────────────────┘  │
│                              │
│  💡 你的数据仅存储在本设备中   │
│     无需联网，绝对隐私安全      │
└──────────────────────────────┘
```

**交互说明**：
- **昵称**：必填，1-12 个字符，用于首页顶部问候语（如「你好，小明」）
- **头像**：可选，提供 6 个预设 emoji 头像供选择（👤 🐱 🐶 🐼 🦊 🐰），默认 👤
- **PIN 码**：可选，4-6 位纯数字。如果跳过，下次打开无需验证；如果设置，则每次打开需输入 PIN
- **CTA 按钮**：点击后创建本地账号档案，进入主界面
- **隐私承诺**：底部说明文案强化用户信任

#### 3.6.2 再次打开 — 登录

**场景一：未设置 PIN 码**

直接进入首页，无需验证。

**场景二：已设置 PIN 码**

```
┌──────────────────────────────┐
│                              │
│        🏦 记账本              │
│                              │
│  ┌────────────────────────┐  │
│  │     👤 小明             │  │  ← 显示账号头像 + 昵称
│  └────────────────────────┘  │
│                              │
│  ┌─┬─┬─┬─┬─┬─┐             │
│  │ │ │ │ │ │ │  ← PIN 输入框  │
│  └─┴─┴─┴─┴─┴─┘             │
│                              │
│  ┌─┬─┬─┬──┐                 │
│  │1│2│3│  │                 │
│  ├─┼─┼─┼──┤                 │
│  │4│5│6│  │  ← 数字键盘      │
│  ├─┼─┼─┼──┤                 │
│  │7│8│9│  │                 │
│  ├─┼─┼─┼──┤                 │
│  │ ← │0│✓ │                 │
│  └─┴─┴─┴──┘                 │
│                              │
│     忘记 PIN？[重置数据]      │  ← 小字提醒
└──────────────────────────────┘
```

**交互说明**：
- PIN 输入框为 4-6 个圆点，每输入一位点亮一个圆点
- 输入完整 PIN 后自动验证，正确则进入首页（带过渡动画）
- 错误时圆点抖动 + 红色闪烁，不清空已输入内容
- 连续错误 5 次锁定 30 秒
- **忘记 PIN**：点击后弹窗确认「重置将删除所有数据，是否继续？」，确认后清除全部 localStorage 数据并回到注册页

- 退出登录按钮（清除当前会话，回到登录页）

## 四、网站整体配色方案

### 4.1 设计理念

整体设计遵循 **Banking / Traditional Finance** 风格，传达**专业、可信赖、稳重**的金融产品氛围。同时提供完整的暗色模式支持，适应不同光照环境下的使用场景。

### 4.2 字体系统

| 用途 | 字体 | 说明 |
|------|------|------|
| 全局正文 | **Plus Jakarta Sans** | 现代几何无衬线体，字怀开阔，可读性极佳 |
| 数字/金额 | Plus Jakarta Sans (tabular-nums) | 启用等宽数字特性，金额对齐不跳动 |
| 后备字体 | -apple-system, BlinkMacSystemFont, Segoe UI, Roboto, sans-serif | 系统默认无衬线后备 |
| 字重 | 300 (Light) / 400 (Regular) / 500 (Medium) / 600 (SemiBold) / 700 (Bold) | 5 级字重覆盖所有排版层级 |

### 4.3 亮色模式（Light Mode）

默认主题，适用于日间使用。

| 令牌 | 色值 | 用途 |
|------|------|------|
| `--primary` | `#0F172A` | 主色调：导航栏、主要按钮、强调文字 |
| `--primary-hover` | `#1E293B` | 主色悬浮态 |
| `--secondary` | `#1E3A8A` | 次要色：链接、次要强调 |
| `--accent` | `#A16207` | 强调色：CTA 按钮、选中状态（暖金色调） |
| `--accent-light` | `#FEF3C7` | 强调色浅底：标签、高亮区域 |
| `--bg` | `#F8FAFC` | 页面背景：浅灰蓝，比纯白更柔和 |
| `--surface` | `#FFFFFF` | 卡片/面板背景：纯白 |
| `--surface-hover` | `#F1F5F9` | 卡片悬浮态 |
| `--border` | `#E2E8F0` | 边框/分割线 |
| `--text` | `#020617` | 正文颜色：近黑色，高对比度 |
| `--text-secondary` | `#475569` | 次要文字：灰色 |
| `--text-tertiary` | `#94A3B8` | 辅助文字：浅灰 |
| `--expense` | `#DC2626` | 支出色：警示红 |
| `--expense-bg` | `#FEF2F2` | 支出背景：淡红底 |
| `--income` | `#059669` | 收入色：成功绿 |
| `--income-bg` | `#ECFDF5` | 收入背景：淡绿底 |

**配色逻辑**：
- 主色调采用**深蓝-灰**系（Slate），传递稳重可靠的金融感受
- 强调色采用**暖金**（Amber），在冷色主调中形成温暖焦点，用于 CTA 按钮引导操作
- 支出用**警示红**（Red-600），收入用**成功绿**（Emerald-600），语义明确，用户无需思考
- 背景采用**微灰**而非纯白，降低长时间使用的视觉疲劳

### 4.4 暗色模式（Dark Mode）

通过 `prefers-color-scheme: dark` 媒体查询自动切换，适用于夜间或弱光环境。

| 令牌 | 色值 | 用途 |
|------|------|------|
| `--primary` | `#1E40AF` | 主色调：深蓝，比亮色模式更亮以保证对比度 |
| `--primary-hover` | `#2563EB` | 主色悬浮态 |
| `--secondary` | `#3B82F6` | 次要色 |
| `--accent` | `#059669` | 强调色：翠绿，暗色下暖金会刺眼，改用冷绿 |
| `--accent-light` | `#064E3B` | 强调色浅底 |
| `--bg` | `#0F172A` | 页面背景：深蓝灰 |
| `--surface` | `#1E293B` | 卡片/面板背景 |
| `--surface-hover` | `#273449` | 卡片悬浮态 |
| `--border` | `rgba(255,255,255,0.08)` | 边框：白色半透明 |
| `--text` | `#F1F5F9` | 正文颜色：浅灰白 |
| `--text-secondary` | `#94A3B8` | 次要文字 |
| `--text-tertiary` | `#64748B` | 辅助文字 |
| `--expense` | `#EF4444` | 支出色：稍亮的红 |
| `--expense-bg` | `#450A0A` | 支出背景：深红底 |
| `--income` | `#10B981` | 收入色：稍亮的绿 |
| `--income-bg` | `#052E16` | 收入背景：深绿底 |

**暗色模式设计原则**：
- 亮色主色 `#0F172A`（近黑）→ 暗色主色 `#1E40AF`（蓝），保持品牌识别的同时确保可读性
- 强调色从暖金 `#A16207` 切换为冷绿 `#059669`，避免暗背景下暖色造成的视觉疲劳
- 阴影在暗色模式下加深，以保证卡片层次感
- 所有颜色对均通过 WCAG 2.1 AA 级对比度验证（≥ 4.5:1）

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 用户难以坚持记账 | 留存低 | 极简交互降低摩擦；后续引入打卡/提醒机制 |
| 分类不满足个性化需求 | 用户不买账 | MVP 即支持自定义分类 |
| 竞品成熟（随手记、钱迹等） | 获客难 | 坚持极简差异化，不做大而全 |
| 收入模型不清晰 | 商业化难 | V3.0 后探索会员制或高级功能付费 |
| 忘记 PIN 码导致数据不可访问 | 用户数据丢失 | 提供「重置数据」入口，明确告知后果；V1.1 引入生物识别作为备用方案 |
| 隐私安全顾虑 | 用户不信任 | 数据完全本地化，零网络请求，代码开源可审计 |
